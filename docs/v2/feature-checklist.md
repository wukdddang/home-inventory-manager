# 기능 체크리스트 v2 (Home Inventory Manager)

**버전**: v2 — 프론트 구현 피드백 반영 (2026-03-26)

**v1 대비 주요 변경**:
- Consumption, WasteRecord → InventoryLog 섹션으로 통합
- ShoppingList 섹션 제거, ShoppingListItem 섹션에 병합
- 장보기 제안(Shopping Suggestions) 기능 추가
- 구매 → 재고 자동 반영 기능 추가
- 소비·폐기 처리 기능 명시

엔티티 목록·관계는 [er-diagram.md v2](./er-diagram.md), 필드 상세는 [entity-logical-design.md v2](./entity-logical-design.md)와 동기화합니다.
**v1 원본**: [v1/feature-checklist.md](../v1/feature-checklist.md)

완료한 항목은 `- [x]`로 표시해 관리하세요.

## InventoryLog (재고 변경 이력) — v2 통합

> **v2 변경**: v1의 Consumption(소비 기록) + WasteRecord(폐기 기록)을 InventoryLog로 통합.
> type: 'in' | 'out' | 'adjust' | 'waste'

- [ ] 사용자는_재고_변경_이력을_조회한다 (품목별·기간별)
- [ ] 사용자는_소비_기록을_등록한다 (type='out', 재고가_감소한다) **(v2: Consumption에서 통합)**
- [ ] 사용자는_폐기_기록을_등록한다 (type='waste', reason 입력, 재고가_감소한다) **(v2: WasteRecord에서 통합)**
- [ ] 사용자는_수량을_수동_조정한다 (type='adjust')

---

## ShoppingListItem (장보기 항목) — v2 변경

> **v2 변경**: ShoppingList(부모) 제거. ShoppingListItem이 Household에 직접 연결. checked 제거 (구매 완료 시 행 삭제).

- [ ] 사용자는_장보기에_항목을_추가한다 (Household 직접 연결)
- [ ] 사용자는_장보기_항목_목록을_조회한다
- [ ] 사용자는_장보기_항목을_수정·삭제한다
- [ ] 사용자는_장보기_항목을_구매_완료_처리한다 (재고 반영 + 행 삭제) **(v2: checked 대체)**
- [ ] 시스템은_부족_품목을_장보기에_자동_제안한다 (Shopping Suggestions) **(v2 추가)**
- [ ] 시스템은_유통기한_임박_품목을_장보기에_자동_제안한다 (Shopping Suggestions) **(v2 추가)**
