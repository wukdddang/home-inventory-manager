# E2E 유즈케이스 시나리오

**최초 작성**: 2026-04-01
**갱신**: 2026-04-03 — v2.7 가전/설비 관리 유즈케이스 추가
**기준**: feature-checklist.md v2.7, api-connection-checklist.md

백엔드 + 프론트엔드 통합 E2E 테스트를 위한 사용자 유즈케이스 시나리오입니다.
각 항목은 테스트 케이스명으로 바로 사용할 수 있도록 "~한다" 형태로 작성했습니다.

범례:

- **검증**: `[ ]` 미검증 | `[x]` 검증 완료 | `[!]` 이슈 발견
- **UI**: `O (`컴포넌트명`)` 구현됨 | `X` 미구현 (관련 컴포넌트 없음) | `-` UI 불필요 (백엔드/인프라)

## 시나리오 실행 순서 (권장)

실제 E2E 테스트 시 아래 순서로 진행하면 데이터 의존성이 자연스럽게 충족됩니다:

```
UC-01 (가입/인증)
  → UC-02 (비밀번호 변경)
  → UC-03 (거점/방/가구/보관장소)
    → UC-04 (카탈로그)
      → UC-05 (재고 등록)
        → UC-06 (구매/로트)
          → UC-07 (소비/폐기/조정)
        → UC-08 (장보기)
      → UC-10 (알림 설정/조회)
    → UC-11 (가전/설비) — v2.7
  → UC-09 (멤버 초대/협업) — 별도 사용자 계정 필요
  → UC-13 (토큰 갱신) — 독립 실행 가능
  → UC-12 (거점 삭제) — 마지막에 실행
```
