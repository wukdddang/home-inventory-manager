# 모바일 E2E 유즈케이스 시나리오

**최초 작성**: 2026-04-02
**기준**: use-cases.md (데스크탑), feature-checklist.md v2.5+, v2.8 아키텍처 반영
**뷰포트**: < 768px (모바일) / 768–1024px (태블릿) — `useDeviceLayout` 기준

모바일 화면 전용 E2E 테스트를 위한 사용자 유즈케이스 시나리오입니다.
데스크탑과 완전히 다른 UI 컴포넌트를 사용하는 구간만 별도로 작성했습니다.

범례:

- **검증**: `[ ]` 미검증 | `[x]` 검증 완료 | `[!]` 이슈 발견
- **UI**: `O (`컴포넌트명`)` 구현됨 | `X` 미구현 | `-` UI 불필요

---

## 모바일 전용 UI 구조

```
MobileShell
├── MobileHeader        — 거점 드롭다운 + 알림 벨
├── Main Content        — 페이지별 모바일 패널
│   ├── DashboardMobilePanel
│   ├── InventoryHistoryMobilePanel
│   └── SettingsMobilePanel
└── BottomNav           — 홈 | 이력 | 설정 (3탭)
```

**모바일 전용 공통 컴포넌트**:
- `BottomSheet` — 하단 시트 (framer-motion 스프링 애니메이션)
- `QuantityStepper` — 터치 친화적 수량 조절 (44px 최소 터치 영역)
- `SwipeableCard` — 스와이프 카드 (정의됨, 미사용)

## 데스크탑 전용 기능 (모바일 미지원)

아래 기능은 모바일에서 별도 UI를 제공하지 않으며, 데스크탑에서만 사용 가능하다:

| 기능 | 데스크탑 컴포넌트 | 모바일 비고 |
|------|-------------------|-------------|
| 구매 관리 (목록/등록/로트) | `PurchaseRegister`, `PurchaseLots` | 모바일 전용 UI 없음 (구매 탭 미노출) |
| 카탈로그 관리 (카테고리/품목/용량·포장/단위) | `CatalogBrowserModal`, `CategoryManageModal` 등 | 설정에서 "데스크탑에서 이용" 안내 |
| 멤버 초대 및 역할 관리 | `HouseholdMembershipSettings` | 모바일 미지원 |
| 비밀번호 변경 | `AccountSecuritySettings` | 모바일 미지원 |
| 집 구조도 편집 (2D 플로우) | `HouseStructureFlow` | 모바일 미지원 |
| 방/가구/보관장소 CRUD | `DashboardRooms`, `DashboardPlacements` | 모바일 미지원 |
| 재고 스프레드시트 뷰 | `ItemsSpreadsheet` | 카드 목록으로 대체 |
| 재고 상세 드로어 (수량 직접 조정) | `ItemDetailDrawer` | 바텀시트 액션으로 대체 |
| 거점 유형 관리 | `HouseholdKindsSettings` | 모바일 미지원 |
| 만료 알림 규칙 관리 | `NotificationSettings` (상세) | 기본 토글만 지원 |
