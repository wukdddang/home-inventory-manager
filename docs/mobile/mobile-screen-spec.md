### 4.2 FCM 토큰 등록 흐름

```
"푸시 알림 받기" 토글 ON 시:
  1. 브라우저 Notification.requestPermission()
     ├─ granted → Firebase getToken() → 서버 등록 → 토글 ON 유지
     ├─ denied  → 토글 OFF 복원 + "브라우저 설정에서 알림을 허용해주세요" 안내
     └─ default → 토글 OFF 복원

"푸시 알림 받기" 토글 OFF 시:
  1. 서버에서 FCM 토큰 삭제 요청
  2. 토글 OFF 반영
```

## 5. 태블릿 분기 (768px ~ 1024px)

태블릿은 모바일 레이아웃을 기본으로 하되, 화면 폭을 활용한 조정:

| 요소 | 모바일 (~768px) | 태블릿 (768px~1024px) |
|------|-----------------|----------------------|
| 재고 카드 | 1열 | 2열 그리드 |
| 이력 타임라인 | 1열 | 1열 (카드 폭 넓게) |
| 바텀시트 | 전체 폭 | `max-w-md` 중앙 정렬 |
| 장보기 목록 | 바텀시트 | 사이드 패널 가능 |

**체크리스트:**
- [x] 재고 카드 그리드 `grid-cols-1 md:grid-cols-2` 적용
- [x] 바텀시트 태블릿 `max-w-md mx-auto` 적용
- [x] 이력 카드 태블릿 폭 조정 (`md:max-w-2xl`)

## 6. 접근성 및 UX 고려사항

| 항목 | 기준 |
|------|------|
| 터치 타겟 | 최소 44×44px |
| 폰트 최소 크기 | 14px (iOS 줌 방지를 위해 input은 16px) |
| 색상 대비 | WCAG AA 기준 (4.5:1) |
| 스와이프 대안 | 카드 탭 시 상세에서 버튼으로도 동작 가능 |
| 로딩 상태 | 스켈레톤 카드 표시 |
| 빈 상태 | 등록된 재고 없음 안내 + 데스크탑 유도 |
| safe-area | `env(safe-area-inset-bottom)` 하단 탭에 적용 |
| 100dvh | `100vh` 대신 `100dvh` 사용 (모바일 주소창 대응) |

**체크리스트:**
- [x] 모든 버튼/탭 터치 타겟 44×44px 이상 확인
- [x] input 요소 `font-size: 16px` (iOS 줌 방지)
- [x] `safe-area-inset-bottom` 하단 탭에 적용
- [x] `100dvh` 사용 확인
- [x] 빈 상태 UI 구현 (재고 없음 / 이력 없음)
- [x] 스켈레톤 로딩 상태 구현 (`MobileSkeleton.component.tsx`)
