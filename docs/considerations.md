# 추가로 고려할 만한 기능

생각나는 기능을 모두 만들고 싶을 때 참고할 **추가 기능 후보** 목록입니다.  
엔티티·API·비즈니스 로직 확장 시 여기에 있는 항목을 반영할 수 있습니다.

---

## 1. 이미 목록에 있는 확장 (Recipe, Brand, Photo 등)

| 구분 | 내용 | 비고 |
|------|------|------|
| **Recipe** | 레시피 등록, 재료와 연결 → 소비 시 “이 레시피로 사용” 기록 | Consumption과 연결, 재료 소진 추적 |
| **Brand** | 상품 브랜드 마스터 (예: CJ, 롯데) | Product에 brandId FK |
| **Supplier** | 구매처/공급처 (마트, 쇼핑몰 이름) | Purchase에 supplierId, 단가 비교 |
| **Photo** | 영수증 사진, 제품 사진 첨부 | Purchase·Product에 연결, Multer/S3 |
| **Integration** | 카카오톡·이메일·푸시 등 알림 채널 | Notification 전송 수단 확장 |

---

## 2. 재고·물품 쪽

| 구분 | 내용 | 비고 |
|------|------|------|
| **반복 구매 알림** | “이 품목은 보통 N주마다 산다” → 주기 알림 | 소비 패턴 기반 리마인더 |
| **자산 가치** | 비소모품(전자제품, 가구) 구매가·현재 추정가 | 감가상각·보험 대비 |
| **A/S·보증 기간** | 구매일·보증 만료일 (비소모품) | PurchaseBatch와 유사하나 “수리/폐기” 관점 |
| **최소/최대 재고** | minStockLevel 외에 maxStockLevel (과재고 방지) | InventoryItem 확장 |
| **바코드 스캔** | 바코드로 상품/재고 등록·조회 | Product.barcode 활용, 스캐너 연동 |
| **위치별 재고 요약** | 보관 장소별 수량·금액 요약 | StorageLocation 기준 집계 API |

---

## 3. 재무·예산·가계부·구독 쪽

| 구분 | 내용 | 비고 |
|------|------|------|
| **고정 지출** | 월세, 관리비, 통신비, 보험 등 반복 지출 등록 | RecurringExpense 엔티티 (RecurringIncome과 대칭) |
| **예정 지출** | 공과금 납부일, 대출 상환일, 카드 결제일 | 일정 + 예상 금액 |
| **예산 설정** | 월별 총예산, 카테고리별 예산 (식비, 교통비 등) | Budget 엔티티, 지출 대비 알림 |
| **가계부** | 수입/지출·잔액·예산을 한 화면에 | Account + Transaction + RecurringIncome/Expense + Budget 조합. [가계부·구독 연동 방안](./ledger-and-subscription.md) 참고. |
| **구독 내역** | 넷플릭스, 멤버십 등 반복 결제 등록·알림 | Subscription 또는 RecurringExpense 확장. 수동 입력 외 **오픈뱅킹·이메일 파싱** 등으로 입력 부담 줄이기 → [가계부·구독 연동 방안](./ledger-and-subscription.md) 참고. |
| **대출·부채** | 빌린 돈, 갚을 날, 이자 | Liability/Debt 엔티티, Account와 연동 |
| **보증금·적금** | 미래에 돌려받을 돈, 만기일 | Account 타입 확장 또는 별도 엔티티 |
| **수입/지출 카테고리** | 급여, 부수입, 식비, 교통비 등 (재무용) | Category와 분리하거나 type으로 구분 |
| **다중 통화** | 원화 외 달러, 유로 등 | Account·Purchase에 currency, 환율(선택) |

## 4. 장보기·소비 쪽

| 구분 | 내용 | 비고 |
|------|------|------|
| **장보기 리스트 템플릿** | 자주 쓰는 리스트 저장 후 복사 | ShoppingList 복제 또는 Preset |
| **예상 금액** | 장보기 리스트 항목별 예상 단가·합계 | ShoppingListItem에 estimatedPrice |
| **구매처 추천** | 품목별 과거 구매처·단가 히스토리 | Purchase + Supplier 기반 |
| **소비 메모/사진** | 소비 시 메모·사진 (레시피 결과 등) | Consumption 확장 |

---

## 5. 알림·리포트 쪽

| 구분 | 내용 | 비고 |
|------|------|------|
| **알림 채널 설정** | 앱/이메일/카카오 중 선택, 품목/알림 유형별 설정 | Notification + Integration |
| **주간/월간 요약 리포트** | 지난 N일 재고 변동, 지출, 만료 예정 요약 | ReportPreset + 집계 API |
| **목표 설정** | “이번 달 식비 50만 원 이하” 등 | Budget + 알림 |
| **만료 알림 다단계** | 7일 전, 3일 전, 당일 등 여러 시점 | ExpirationAlertRule을 여러 개 허용 (이미 가능) |

---

## 6. 협업·편의

| 구분 | 내용 | 비고 |
|------|------|------|
| **가족/공유 그룹 초대 링크** | invitedCode로 가입 없이 초대 | Household.invitedCode 활용 |
| **역할/권한** | 리스트 수정만 가능, 재고 수정 불가 등 | HouseholdMember.role 세분화 |
| **활동 로그** | “누가 언제 무엇을 추가/수정했는지” | AuditLog 엔티티 (User, targetType, targetId) |
| **데이터 내보내기/가져오기** | CSV, 엑셀 백업·복원 | 배치 API 또는 스크립트 |
| **백업/스냅샷** | DB 또는 JSON 스냅샷 저장·복원 | 운영·이관용 |

---

## 7. 기타

| 구분 | 내용 | 비고 |
|------|------|------|
| **카테고리 계층** | 대분류 > 중분류 > 소분류 (이미 설계에 parentId 있음) | Category 자기 참조 활성화 |
| **태그 검색/필터** | 목록 화면에서 태그로 필터, 통계 | Tag + Query |
| **다국어** | 카테고리·단위 등 마스터 다국어 (선택) | i18n, 나중에 필요 시 |
| **모바일 앱 / PWA** | 프론트 별도 프로젝트에서 이 API 사용 | 인증·CORS 정리 |

---

## 정리

- **우선 넣기 좋은 것**: Recipe, Brand, Supplier, Photo, 고정/예정 지출, 예산 설정, 반복 구매 알림, 장보기 템플릿·예상 금액.
- **재무 확장**: 대출·부채, 보증금·적금, 수입/지출 카테고리, 다중 통화.
- **운영·편의**: 초대 링크, 활동 로그, 내보내기/가져오기, 백업.

필요한 것부터 엔티티·기능 목록(예: [feature-checklist.md](./feature-checklist.md), [entity-logical-design.md](./entity-logical-design.md))에 반영해 가면 됩니다.
