# 프론트 UI 개발 진행 현황

**목적**: 백엔드 API 연동 여부와 무관하게, **화면·인터랙션·플로우**가 얼마나 갖춰졌는지 확인하기 위한 체크리스트입니다.  
데이터는 로컬스토리지·목 시드로도 동작하며, 여기서는 “그걸로 **어떤 UI가 완성됐는지**”만 다룹니다.

도메인 용어·ERD 정본은 루트 [ER 다이어그램](../../docs/er-diagram.md), [기능 체크리스트](../../docs/feature-checklist.md), [엔티티 논리적 설계](../../docs/entity-logical-design.md)를 참고하면 됩니다.

**표기**: `- [x]` = UI(및 클라이언트 상태)까지 구현됨 · `- [ ]` = 화면·폼·흐름 없음 또는 일부만 있음

**범위 밖**: REST/GraphQL 연동, 서버 인증, 배포·인프라 — 본 문서에 포함하지 않습니다.

## 1. 한눈에 보는 요약

| 구역 | 진행도 (UI) |
|------|-------------|
| 앱 쉘·라우트 | 핵심 경로·가드·로딩·페이지 진입 시 progress 막대(`AppLoadingState`) (§2) |
| 대시보드 | 거점·방·구조도·배치·재고·카탈로그 빠른 추가까지 **주요 플로우 연결됨** (§2) |
| 설정 | 카탈로그·알림·계정·보안(모달)·거점 멤버십(초대·역할·제거, 목)까지 연결됨 (§2) |
| 구매·로트 | `/purchases` 등록·목록·임박·`/mock` 시드 (§2.4) |
| 재고 이력 | `/inventory-history` 타임라인·거점 필터·`him-inventory-ledger` (§2.6) |
| 등록 경로 안내 | **메인**(방·보관 장소·재고 한꺼번에) vs **구매·로트**(먼저 영수증·로트만) — [`screens-overview.md`](./screens-overview.md) §「재고·구매 등록」 |
| 알림 센터 | 헤더 벨 아이콘·모달: 수신 목록·읽음·삭제·미읽음 카운트 (§2.10) |
| 아직 없는 UI | 품목별 만료 규칙 편집 등 (§3·§4). 구매→재고 자동 반영은 백엔드에서 처리, 장보기 전용 라우트는 모달+패널로 대체 확정 |

## 2. 구현된 UI (체크리스트)

### 2.1 라우트·쉘

- [x] `/`, `/login`, `/signup`, `/dashboard`, `/purchases`, `/inventory-history`, `/settings` 및 `/mock/…` 대응
- [x] `AppShell`, `AuthGuard`, 로딩·화면 전환(대시보드·설정·구매 패널 등 framer-motion)
- [x] **페이지 진입 로딩** — 대시보드(mock `list` 지연)와 동일하게 느껴지도록 설정·구매 Context에 최소 로딩 시간(`APP_PAGE_MIN_LOADING_MS`) 적용 → `AppLoadingState` progress 막대 노출

### 2.2 대시보드 (`/dashboard`)

- [x] **거점** — 탭 전환, 추가·삭제, 유형 관리 모달, 탭 선택 하이라이트 애니메이션
- [x] **방** — 탭·추가·이름 수정·삭제, 탭 하이라이트 애니메이션
- [x] **가구·보관 장소** — 배치 편집, 방 직속/가구 하위 슬롯 UI ([집 구조 UX](./house-structure-feature.md))
- [x] **구조도·플로우** — 방·보관 장소·가구·재고 흐름 조회·편집 (`HouseStructureFlow` 등)
- [x] **재고** — 등록·목록·수량 등, 카탈로그 품목·용량과 연계
- [x] **카탈로그 빠른 추가** — 카테고리·품목·용량·포장 모달, 재고 추가 패널·모달 모션
- [x] **뷰 모드** — 구조도 / 목록 등 전환
- [x] **로트·임박(재고 목록)** — `him-purchases`에서 동일 `inventoryItemId`로 연결된 구매 로트 요약·뱃지(구조도 옆 패널·표 모드)
- [x] **소비·폐기** — 재고 행에서 수량·메모·(폐기 시) 사유 입력 모달 → 수량 감소 + `him-inventory-ledger` (`out` / `waste`)
- [x] **재고 이력 바로가기** — 조회 모드 설명 아래 `/inventory-history` 링크

### 2.3 설정 (`/settings`)

- [x] **상품 카탈로그** — 카테고리·단위·품목·변형 설정 (`CatalogModals`와 동일 도메인)
- [x] **알림 선호(설정 화면)** — 유통기한·장보기·부족 재고 관련 토글·숫자·스코프
- [x] **계정·보안** — 카드 요약(이메일·표시 이름·인증 뱃지) + `FormModal`로 프로필·이메일 인증(데모)·비밀번호 변경; `/mock`은 `him-mock-settings-account` 목 계정
- [x] **거점 멤버십** — 멤버 목록, 역할 뱃지·팝오버로 변경, `FormModal`로 멤버 추가(초대 폼), `AlertModal`로 제거(마지막 단일 소유자 보호); 거점·멤버는 목/로컬 상태

### 2.4 구매·로트 (`/purchases`)

- [x] **구매 기록·로트 목록** — 거점 탭, 테이블(품목·구매일·금액·로트·임박 뱃지·삭제)
- [x] **구매 등록 모달** — 재고 연결(선택)·로트 다건·`him-purchases` 저장
- [x] **`/mock` 목 시드** — 로컬에 거점·구매가 없을 때 대시보드와 동일 거점 시드 + 예시 구매·로트(날짜는 «오늘» 기준 생성)

> **사용자 안내**: 이미 보관 장소에 둔 뒤 입력할 때는 **메인**에서 재고를 넣는 편이 한 번에 맞고, 장만 하고 보관 장소 정리는 나중이면 **구매·로트**에서 로트만 먼저 적어 두면 됩니다(재고 연결은 선택). 요약은 [`screens-overview.md`](./screens-overview.md) §「재고·구매 등록」 참고.

> **다음 단계(예시)**: 구매 등록만으로 재고 수량·로트가 자동 반영되게 하거나, 재고 **상세 패널**에서 동일 정보를 모아 보기.

### 2.7 장보기 (`him-shopping-list`)

- [x] **저장·거점별 목록** — 로컬스토리지, 선택 거점 기준 필터

- [x] ~~**쇼핑 중 체크리스트**~~ — `구매 완료` 흐름으로 대체 (최종 결정)

- [x] **재고 등록 폼** — 카탈로그 경로·구매 경로 모두 수량 옆에 최소 재고 입력 (가로 2열)

- [x] **거점 생성 온보딩 마법사** — 거점 추가 직후 모달 팝업. 4단계(방 추가 → 가구 → 보관 장소 → 완료) 스텝 위자드. framer-motion 전환 애니메이션. 각 단계에서 즉시 추가 가능, 건너뛰기 지원.

### 2.10 알림 센터

- [x] **읽음 처리** — 개별 읽음, 전체 읽음. 미읽음 항목은 뱃지·점 표시.
- [x] **삭제** — 개별 삭제, 전체 삭제.
- [x] **미읽음 카운트** — 헤더 벨 아이콘 위 숫자 뱃지로 미읽음 수 표시.
- [x] **mock 시드** — `/mock` 모드에서 예시 알림 7건 자동 시드(`him-notifications`).

## 3. UI가 없거나 설정/문구 수준인 영역

제품 스펙(ERD) 대비 **사용자가 조작할 화면이 아직 없는 것**입니다. 항목을 만들면 `[x]`로 바꾸고 §2로 옮깁니다.

- [x] **품목별 만료 알림 일수** — 설정 > 알림 > 유통기한 카드 하단에 품목별 오버라이드 목록 (§2.3)
- [ ] **리포트 프리셋·태그** — 관련 화면 없음 (우선순위 낮음 — 리포트 화면 자체가 없고, 태그는 카탈로그 카테고리로 대체 가능. ERD에서도 "줄이기/미루기"로 분류)
