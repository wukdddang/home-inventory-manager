# 프론트 UI 개발 진행 현황

**목적**: 백엔드 API 연동 여부와 무관하게, **화면·인터랙션·플로우**가 얼마나 갖춰졌는지 확인하기 위한 체크리스트입니다.  
데이터는 로컬스토리지·목 시드로도 동작하며, 여기서는 “그걸로 **어떤 UI가 완성됐는지**”만 다룹니다.

도메인 용어·ERD 정본은 루트 [ER 다이어그램](../../docs/er-diagram.md), [기능 체크리스트](../../docs/feature-checklist.md), [엔티티 논리적 설계](../../docs/entity-logical-design.md)를 참고하면 됩니다.

**표기**: `- [x]` = UI(및 클라이언트 상태)까지 구현됨 · `- [ ]` = 화면·폼·흐름 없음 또는 일부만 있음

**범위 밖**: REST/GraphQL 연동, 서버 인증, 배포·인프라 — 본 문서에 포함하지 않습니다.

| 구역 | 진행도 (UI) |
|------|-------------|
| 앱 쉘·라우트 | 핵심 경로·가드·로딩 있음 (§2) |
| 대시보드 | 거점·방·구조도·배치·물품·카탈로그 빠른 추가까지 **주요 플로우 연결됨** (§2) |

## 2. 구현된 UI (체크리스트)

### 2.1 라우트·쉘

- [x] `/`, `/login`, `/signup`, `/dashboard`, `/settings` 및 `/mock/…` 대응
- [x] `AppShell`, `AuthGuard`, 로딩·화면 전환(일부 framer-motion)

### 2.2 대시보드 (`/dashboard`)

- [x] **거점** — 탭 전환, 추가·삭제, 유형 관리 모달, 탭 선택 하이라이트 애니메이션
- [x] **방** — 탭·추가·이름 수정·삭제, 탭 하이라이트 애니메이션
- [x] **가구·보관 칸** — 배치 편집, 방 직속/가구 하위 슬롯 UI ([집 구조 UX](./house-structure-feature.md))
- [x] **구조도·플로우** — 방·칸·가구·물품 흐름 조회·편집 (`HouseStructureFlow` 등)
- [x] **물품(재고)** — 등록·목록·수량 등, 카탈로그 품목·용량과 연계
- [x] **카탈로그 빠른 추가** — 카테고리·품목·용량·포장 모달, 물품 추가 패널·모달 모션
- [x] **뷰 모드** — 구조도 / 목록 등 전환

### 2.3 설정 (`/settings`)

- [x] **상품 카탈로그** — 카테고리·단위·품목·변형 설정 (`CatalogModals`와 동일 도메인)
- [x] **알림 선호(설정 화면)** — 유통기한·장보기·부족 재고 관련 토글·숫자·스코프
- [x] **그룹 멤버** — 멤버 목록 표시(목 데이터)

### 2.4 공통 UI

- [x] `FormModal`, `MotionModalLayer`, `AlertModal` 패턴
- [x] 카탈로그 품목 이미지 업로드(용량 제한·미리보기)

## 3. UI가 없거나 설정/문구 수준인 영역

제품 스펙(ERD) 대비 **사용자가 조작할 화면이 아직 없는 것**입니다. 항목을 만들면 `[x]`로 바꾸고 §2로 옮깁니다.

- [ ] **계정·보안 UI** — 비밀번호 변경, 프로필, 이메일 인증 안내/완료 화면 등(로그인·가입 폼만 있음)
- [ ] **거점 멤버십 UI** — 초대, 역할 변경, 멤버 제거 등(멤버 **목록 표시**만 있음)
- [ ] **구매·유통기한 로트** — 구매 입력, 로트 목록, 임박/만료 전용 뷰
- [ ] **소비·폐기·재고 이력** — 기록 입력·타임라인·필터
- [ ] **장보기** — 리스트 CRUD, 항목 체크, 부족 품목 자동 반영 UI
- [ ] **알림 센터** — 수신 목록, 읽음 처리(설정 토글만 있음)
- [ ] **품목별 만료 알림 일수** — 품목 단위 규칙 편집 UI(전역/템플릿 설정만 있음)
- [ ] **최소 재고(minStock)** — 물품 폼·목록에 임계값 입력·표시
- [ ] **집 구조 전용 화면** — 거점 안에 통합된 편집만 있음; 별도 “구조만 다루는” 설정 탭·마법사 등은 없음
- [ ] **리포트 프리셋·태그** — 관련 화면 없음

## 4. UI 개발 로드맵 (다음에 그릴 화면)

우선순위는 팀에서 조정. **화면·컴포넌트·클라이언트 상태** 기준입니다.

### 4.1 재고·구매·장보기 경험

- [ ] 구매 등록 + 로트(유통기한) 입력·목록 UI
- [ ] 물품 상세/목록에 로트·임박 표시(디자인에 맞게 카드·뱃지 등)
- [ ] `minStockLevel` 입력 필드 + 목록에서 부족 강조
- [ ] 소비·폐기 단발 입력 + 간단 이력 리스트
- [ ] 장보기 리스트 페이지(또는 패널) + 항목 추가·체크·삭제
- [ ] 알림 센터(목록·읽음) — 설정과 분리된 화면

### 4.2 설정·계정 보강

- [ ] 품목별 만료 알림 일수 편집(품목 선택 → 일수)
- [ ] 계정: 프로필·비밀번호 변경·(필요 시) 이메일 인증 UX
- [ ] 거점: 멤버 초대·역할·제거 플로우 UI

### 4.3 구조·카탈로그 UI 디테일

- [ ] 카탈로그: 품목·변형 **수정·삭제** 흐름을 설정과 동일 수준으로 정리
- [ ] 가구 배치 ↔ 상품(종류) 연결 UI — ERD상 선택 필드 반영
- [ ] (선택) 집 구조만 다루는 전용 구역·온보딩

### 4.4 품질 (UI 관점)

- [ ] 핵심 플로우 키보드·스크린리더 점검
- [ ] (필요 시) 문자열 분리·locale
- [ ] (필요 시) 주요 플로우 E2E
