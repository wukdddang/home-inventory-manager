# 프론트 UI 개발 진행 현황

**목적**: 백엔드 API 연동 여부와 무관하게, **화면·인터랙션·플로우**가 얼마나 갖춰졌는지 확인하기 위한 체크리스트입니다.  
데이터는 로컬스토리지·목 시드로도 동작하며, 여기서는 “그걸로 **어떤 UI가 완성됐는지**”만 다룹니다.

도메인 용어·ERD 정본은 루트 [ER 다이어그램](../../docs/er-diagram.md), [기능 체크리스트](../../docs/feature-checklist.md), [엔티티 논리적 설계](../../docs/entity-logical-design.md)를 참고하면 됩니다.

**표기**: `- [x]` = UI(및 클라이언트 상태)까지 구현됨 · `- [ ]` = 화면·폼·흐름 없음 또는 일부만 있음

**범위 밖**: REST/GraphQL 연동, 서버 인증, 배포·인프라 — 본 문서에 포함하지 않습니다.

## 1. 한눈에 보는 요약

| 구역 | 진행도 (UI) |
|------|-------------|
| 앱 쉘·라우트 | 핵심 경로·가드·로딩·페이지 진입 시 progress 막대(`AppLoadingState`) (§2) |
| 대시보드 | 거점·방·구조도·배치·물품·카탈로그 빠른 추가까지 **주요 플로우 연결됨** (§2) |
| 설정 | 카탈로그·알림·계정·보안(모달)·거점 멤버십(초대·역할·제거, 목)까지 연결됨 (§2) |
| 구매·로트 | `/purchases` 등록·목록·임박·`/mock` 시드 (§2.4) |
| 재고 이력 | `/inventory-history` 타임라인·거점 필터·`him-inventory-ledger` (§2.6) |
| 아직 없는 UI | 장보기 리스트, 알림함, **구매 등록 시 재고 수량 자동 반영**, 품목별 만료 규칙 편집 등 (§3·§4) |

## 2. 구현된 UI (체크리스트)

### 2.1 라우트·쉘

- [x] `/`, `/login`, `/signup`, `/dashboard`, `/purchases`, `/inventory-history`, `/settings` 및 `/mock/…` 대응
- [x] `AppShell`, `AuthGuard`, 로딩·화면 전환(대시보드·설정·구매 패널 등 framer-motion)
- [x] **페이지 진입 로딩** — 대시보드(mock `list` 지연)와 동일하게 느껴지도록 설정·구매 Context에 최소 로딩 시간(`APP_PAGE_MIN_LOADING_MS`) 적용 → `AppLoadingState` progress 막대 노출

### 2.2 대시보드 (`/dashboard`)

- [x] **거점** — 탭 전환, 추가·삭제, 유형 관리 모달, 탭 선택 하이라이트 애니메이션
- [x] **방** — 탭·추가·이름 수정·삭제, 탭 하이라이트 애니메이션
- [x] **가구·보관 칸** — 배치 편집, 방 직속/가구 하위 슬롯 UI ([집 구조 UX](./house-structure-feature.md))
- [x] **구조도·플로우** — 방·칸·가구·물품 흐름 조회·편집 (`HouseStructureFlow` 등)
- [x] **물품(재고)** — 등록·목록·수량 등, 카탈로그 품목·용량과 연계
- [x] **카탈로그 빠른 추가** — 카테고리·품목·용량·포장 모달, 물품 추가 패널·모달 모션
- [x] **뷰 모드** — 구조도 / 목록 등 전환
- [x] **로트·임박(물품 목록)** — `him-purchases`에서 동일 `inventoryItemId`로 연결된 구매 로트 요약·뱃지(구조도 옆 패널·표 모드)
- [x] **소비·폐기** — 물품 행에서 수량·메모·(폐기 시) 사유 입력 모달 → 수량 감소 + `him-inventory-ledger` (`out` / `waste`)
- [x] **재고 이력 바로가기** — 조회 모드 설명 아래 `/inventory-history` 링크

### 2.3 설정 (`/settings`)

- [x] **상품 카탈로그** — 카테고리·단위·품목·변형 설정 (`CatalogModals`와 동일 도메인)
- [x] **알림 선호(설정 화면)** — 유통기한·장보기·부족 재고 관련 토글·숫자·스코프
- [x] **계정·보안** — 카드 요약(이메일·표시 이름·인증 뱃지) + `FormModal`로 프로필·이메일 인증(데모)·비밀번호 변경; `/mock`은 `him-mock-settings-account` 목 계정
- [x] **거점 멤버십** — 멤버 목록, 역할 뱃지·팝오버로 변경, `FormModal`로 멤버 추가(초대 폼), `AlertModal`로 제거(마지막 단일 소유자 보호); 거점·멤버는 목/로컬 상태

### 2.4 구매·유통기한 로트 (`/purchases`)

- [x] **구매 기록·로트 목록** — 거점 탭, 테이블(품목·구매일·금액·로트·임박 뱃지·삭제)
- [x] **구매 등록 모달** — 재고 연결(선택)·로트 다건·`him-purchases` 저장
- [x] **`/mock` 목 시드** — 로컬에 거점·구매가 없을 때 대시보드와 동일 거점 시드 + 예시 구매·로트(날짜는 «오늘» 기준 생성)

> **다음 단계(예시)**: 구매 등록만으로 재고 수량·로트가 자동 반영되게 하거나, 물품 **상세 패널**에서 동일 정보를 모아 보기.

### 2.5 공통 UI

- [x] `FormModal`, `MotionModalLayer`, `AlertModal` 패턴
- [x] 카탈로그 품목 이미지 업로드(용량 제한·미리보기)
- [x] `InventoryLotExpiryBadge` — 물품 목록·표 공용 임박 뱃지

### 2.6 재고 이력 (`/inventory-history`)

- [x] **타임라인** — 최신순, 유형(소비·폐기·입고·조정) 뱃지·Δ수량·잔여·메모·폐기 사유
- [x] **거점 필터** — `him-households`가 비어 있을 때는 대시보드 목 거점 시드로 라벨 보강

## 3. UI가 없거나 설정/문구 수준인 영역

제품 스펙(ERD) 대비 **사용자가 조작할 화면이 아직 없는 것**입니다. 항목을 만들면 `[x]`로 바꾸고 §2로 옮깁니다.

- [ ] **장보기** — 리스트 CRUD, 항목 체크, 부족 품목 자동 반영 UI
- [ ] **알림 센터** — 수신 목록, 읽음 처리(설정 토글만 있음)
- [ ] **품목별 만료 알림 일수** — 품목 단위 규칙 편집 UI(전역/템플릿 설정만 있음)
- [ ] **최소 재고(minStock)** — 물품 폼·목록에 임계값 입력·표시
- [ ] **집 구조 전용 화면** — 거점 안에 통합된 편집만 있음; 별도 “구조만 다루는” 설정 탭·마법사 등은 없음
- [ ] **리포트 프리셋·태그** — 관련 화면 없음

## 4. UI 개발 로드맵 (다음에 그릴 화면)

우선순위는 팀에서 조정. **화면·컴포넌트·클라이언트 상태** 기준입니다.

### 4.1 재고·구매·장보기 경험

- [x] 구매 등록 + 로트(유통기한) 입력·목록 UI — §2.4
- [x] 물품 목록·표에 로트·임박 표시 + 소비·폐기 입력 + 재고 이력 타임라인 — §2.2·§2.6
- [ ] **구매 → 재고 수량·로트 자동 반영**(현재는 UI·저장소만 연결, 수량은 소비·폐기로만 감소)
- [ ] 물품 **전용 상세** 패널(드로어 등)에서 로트·이력 한곳에 모으기
- [ ] `minStockLevel` 입력 필드 + 목록에서 부족 강조
- [ ] 장보기 리스트 페이지(또는 패널) + 항목 추가·체크·삭제
- [ ] 알림 센터(목록·읽음) — 설정과 분리된 화면

### 4.2 설정·계정 보강

- [ ] 품목별 만료 알림 일수 편집(품목 선택 → 일수)
- [x] 계정: 프로필·비밀번호 변경·이메일 인증 UX — §2.3(모달·목/로컬); **서버 인증·실제 메일 발송은 범위 밖**
- [x] 거점: 멤버 초대·역할·제거 플로우 UI — §2.3(목/로컬); **초대 링크·서버 동기화는 범위 밖**

### 4.3 구조·카탈로그 UI 디테일

- [ ] 카탈로그: 품목·변형 **수정·삭제** 흐름을 설정과 동일 수준으로 정리
- [ ] 가구 배치 ↔ 상품(종류) 연결 UI — ERD상 선택 필드 반영
- [ ] (선택) 집 구조만 다루는 전용 구역·온보딩

### 4.4 품질 (UI 관점)

- [ ] 핵심 플로우 키보드·스크린리더 점검
- [ ] (필요 시) 문자열 분리·locale
- [ ] (필요 시) 주요 플로우 E2E
